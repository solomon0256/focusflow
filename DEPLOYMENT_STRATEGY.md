# FocusFlow 部署与分发策略对比

## 1. 方案一：Vercel (推荐用于全球/学术展示)
*   **流程**: 关联 GitHub 仓库 -> 自动构建 -> 分发到全球边缘节点。
*   **优点**: 
    *   **全自动**: 每次提交代码都会自动更新网站。
    *   **极速 (海外)**: 教授在国外访问时速度极快。
    *   **免费**: 个人版完全免费。
*   **缺点**: 
    *   **国内波动**: 中国大陆部分地区访问 Vercel 域名可能存在间歇性变慢。

## 2. 方案二：阿里云 OSS + CDN (推荐用于中国 Android 分发)
*   **流程**: `npm run build` 生成静态文件 -> 上传到 OSS Bucket -> 绑定已备案域名 -> 开启 CDN。
*   **优点**: 
    *   **国内极速**: 阿里云节点遍布全国，APK 下载速度可达兆级。
    *   **合规性**: 配合国内云服务商，更利于长期的 Android 商店审核。
*   **缺点**: 
    *   **成本**: 流量大时会产生少量费用。
    *   **门槛**: 需要域名在国内备案 (ICP)。

---

## 3. 最终建议架构 (The Hybrid Approach)
1.  **代码库**: 维护一套代码。
2.  **主域名 (focusflow.app)**: 托管在 Vercel，用于 PhD 申请和全球展示。
3.  **下载备用域名 (cn.focusflow.app)**: 托管在阿里云，专门存放 APK 和中国区专用落地页。
